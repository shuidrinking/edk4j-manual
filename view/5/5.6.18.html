5.6.18、edk4j配置读取工具
类：com.edk4j.core.util.ConfigPropertiesReader
这个类的方法必须在项目启动后才有效执行，所以做测试时，直接在main方法里调用是会报错的。
前面章节中，讲过了“配置参数怎么获取”，按我们的规范，每组配置项都要有一个特定的配置映射类，然后通过映射类获得配置值。
但是，这个类是在不需要配置映射类的情况下读取配置值的。
方法清单：
public static final String getPropertyValue(String code) 获取配置的key为code的值
public static final String getPropertyValue(String code, String defaultValue) 获取配置的key为code的值，如果配置值不存在时，返回defaultValue
public static final String getPropertyValue(String code, Locale locale) 根据语种locale，获取配置的key为code的值，如果配置值不存在时，返回defaultValue
public static final String getPropertyValue(String code, Object[] placeHolderValue) 获取配置的key为code的值，配置值中的占位符（即{n}）的值，会依次用placeHolderValue中的值替换。
public static final String getPropertyValue(String code, Object[] placeHolderValue, Locale locale) 根据语种locale，获取配置的key为code的值，配置值中的占位符（即{n}）的值，会依次用placeHolderValue中的值替换。
public static final String getPropertyValue(String code, String defaultValue, Object[] placeHolderValue) 获取配置的key为code的值，配置值中的占位符（即{n}）的值，会依次用placeHolderValue中的值替换。
如果配置中没有这个配置时，返回默认值defaultValue
public static final String getPropertyValue(String code, String defaultValue, Object[] placeHolderValue, Locale locale) 根据语种locale，获取配置的key为code的值，配置值中的占位符（即{n}）的值，会依次用placeHolderValue中的值替换。
如果配置中没有这个配置时，返回默认值defaultValue
