99.5、SpringMvc套路下输入输出实体类的规范

99.5.1、入参实体类
（1） 命名 XXXForm或者XXXRequest
（2） 不分页时必须继承com.edk4j.core.web.ioentity.BaseForm
（3） 分页时,必须继承com.edk4j.core.web.ioentity..PageForm
（4） 实体类必须实现Serializable接口,支持序列化
99.5.2、@RequestBody用途的入参类最好加这个注解
使用下面的注解,忽略那些无关字段，在所有输入实体类头上加一下这个注解:
@JsonIgnoreProperties(ignoreUnknown = true)
99.5.3、出参实体类

如果你使用的是spring的开发天路，那么， 必须使用该类作为结果返回类com.edk4j.core.web.ioentity.OperationResult
具体业务中要返回的业务数据实体类,设置到OperationResult的businessDataObject中去

该类被new以后，默认的三个字段如下：
/**
* 返回码
*/
private String resultCode="0";
/**
* 返回类型
*/
private String resultMessage="ok";
/**
* 业务数据对象
*/
Object businessDataObject=null;
开发时，返回的结果数据，需要调用 setBusinessDataObject(Object businessDataObject) 方法设置进去，返回给前台的是json对象

其他值错误，当有错误情况时，前台js自行处理
