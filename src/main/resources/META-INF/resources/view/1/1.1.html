1.1、“edk4j开发平台”是什么
平台概要
全称：“edk4j开发平台”
英文名：enterprise development kit for java
简称：edk4j
简介：零编码或少量编码就可以快速构建并开发应用程序的“低代码”极速Java开发平台。通过图形化、可视化的开发IDE，进行拖拽、组合、调取程序组件的方式来开发应用程序。
edk4j是“低代码极速开发平台”

2014 年，著名的咨询公司Forrester Research提出了“零代码”和“低代码”的概念：只需用很少甚至几乎不需要代码就可以快速开发出系统，并可以将其快速配置和部署的一种技术和工具。

“低代码平台”就是将逻辑功能沉淀为技术组件，通过可视化的开发工具等便捷的形式直接复用，以达到在不写代码或者少写代码的低成本基础上，实现产能最大化。“低代码”、“零代码”是未来软件行业发展的新趋势。

edk4j开发团队与时俱进，推出Java低代码极速开发平台——“edk4j开发平台”。

edk4j定位于“基础设施型平台”
EDK4j
目前市面上已有的java低代码开发平台，例如：JEPasS、JavaFast、Jeeplus、Jeecg等，它们的共同特点是 ：
  1. 组合使用SpringBoot、Mybatis等开源框架；
  2. 提炼出一些比较通用的功能性模块代码，便于项目开发中复用；
  3. 这种所谓的“低代码”仅仅是方便开发过程中省去这些功能的建设，但是这些功能包都有使用局限；
  4. 在他们官方网站上都有使用范围说明，一般都用来做“OA、CRM、CMS、ERP”等后台管理性质的web系统；
  5. 其中的“自动代码生成”，也仅仅是对数据模型的增删改刷操作；
  6. 要使用这些平台做开发，仍然需要手工编写所有代码。
“edk4j开发平台”与Spring、Hibernate等开源框架所处的层面一样，是“基础设施型”的开发平台， 而不是“功能包型”的框架，换句话说，我们拿“edk4j开发平台”能够开发跟他们功能一样的功能包， 而他们反过来就不行，“edk4j开发平台”是开发平台里更底层一级的，没有使用范围的局限性。
